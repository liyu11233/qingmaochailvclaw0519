# 青猫差旅采集工具

本项目用于本机采集青猫差旅、携程商旅、阿里商旅的同航班价格，生成客户可看的 Excel 和销售长截图。

当前版本是本地工具，不是桌面 App，也不是云端 SaaS。真实采集依赖你本人在本机 Chrome 里手动登录三平台，并保持登录窗口打开。

## 当前能力

- 管理采集端：本机打开，负责登录、连接、采集、导出。
- 完整真实采集：国内 10 条 + 国际 10 条航线，三平台同航班比价。
- 导出 Excel：含客户版汇总页、网页截图索引页、留痕页；Excel 不再内嵌截图图片，截图通过索引页追溯。
- 导出销售长截图：把当前批次展示页渲染为一张 PNG，便于直接发给销售或客户查看。
- 运行产物默认保存在 `outputs/current/`，不进入 Git。

## 环境要求

- macOS
- Node.js 18 或更高版本
- npm
- Google Chrome
- 三个平台账号需要你本人手动登录

首次使用：

```bash
npm install
```

如需自定义端口、平台入口或 Chrome 路径：

```bash
cp .env.example .env
```

然后按需修改 `.env`。`.env` 不会提交到 Git。

## 启动方式

### 方式一：双击启动

双击项目根目录的：

```text
启动青猫差旅采集工具.command
```

脚本会进入项目目录，必要时安装依赖，启动本地服务，并打开管理页面。

默认访问地址：

```text
http://localhost:5173
```

关闭启动窗口即可停止本地服务。

### 方式二：终端启动

```bash
npm run dev
```

如果要让 `.env` 生效：

```bash
set -a
source .env
set +a
npm run dev
```

## 日常真实采集流程

1. 启动管理采集端。
2. 点击 `打开登录浏览器`。
3. 在打开的 Chrome 窗口里分别登录：
   - 青猫差旅
   - 携程商旅
   - 阿里商旅
4. 三个平台都登录完成后，不要关闭这个 Chrome 窗口。
5. 回到管理采集端，点击 `连接当前登录窗口`。
6. 看到三平台都可访问后，点击 `开始完整真实采集`。
7. 等待采集完成。完整采集通常需要十几分钟，期间不要重复点击连接或后台探测。采集不会主动把浏览器拉到最前台，但仍需保持登录 Chrome 窗口不要关闭。
8. 完成后点击：
   - `导出 Excel`
   - `销售长截图`

## 导出物在哪里

当前批次导出物在：

```text
outputs/current/
```

常见文件：

- `青猫差旅航班比价-<batch-id>.xlsx`
- `青猫差旅销售长截图-<batch-id>.png`
- `screenshots/<batch-id>/`
- `current-state.json`

这些文件包含真实采集结果、网页截图或本机状态，不上传 GitHub。

## 销售长截图使用方式

1. 把 `青猫差旅销售长截图-<batch-id>.png` 发给销售。
2. 销售可直接打开图片给客户看，也可以放进微信、飞书、PPT 或邮件。
3. 客户需要复核时，再配合 Excel 的网页截图索引或平台入口查询。

销售长截图不能触发采集，也不依赖你的电脑在线。

## GitHub 同步规则

提交前先跑：

```bash
npm test
npm run typecheck
npm run build
```

再检查：

```bash
git status --short
git check-ignore -v outputs .runtime .env node_modules
```

不要提交：

- `outputs/`
- `.runtime/`
- `.env`
- `node_modules/`
- 登录态
- 真实截图
- 导出的 Excel
- 销售长截图 PNG

根目录两个原始 Excel 模板使用 Git LFS 管理，不要改成普通 Git 大文件。

## 常见问题

### 页面打不开

确认本地服务正在运行：

```bash
npm run dev
```

默认地址是 `http://localhost:5173`。如果 `.env` 改过 `PORT`，用新的端口。

### 连接当前登录窗口失败

按顺序检查：

1. 是否通过管理端点击过 `打开登录浏览器`。
2. 三个平台是否都在那个 Chrome 窗口里登录完成。
3. 登录窗口是否仍然打开。
4. 是否有另一个采集或探测操作正在运行。

### 完整真实采集失败

先看管理端错误提示。当前规则是：如果无法凑够国内 10 条、国际 10 条完整样本，本次采集失败并保留上一批可用数据。

不要直接删除 `outputs/current/`。需要排查时先保留现场。

# 青猫差旅航班比价第一版需求冻结稿与实施规划

日期：2026-05-18

## 1. 需求冻结结论

第一版只做航班比价闭环。

目标不是做一个泛比价工具，而是做一个能支持销售现场演示、会后留痕、内部复盘的企业差旅价格证明工具。

第一版范围：

- 国内机票
- 国际机票
- 青猫差旅、携程商旅、阿里商旅三平台对比
- 销售演示页
- 可导出 Excel
- 三平台截图证据保存

第一版不做：

- 酒店价格对比
- 打车价格对比
- 普通 OTA 平台对比
- 账号密码自动登录
- 无人值守定时自动采集

## 2. CEO Review

### 2.1 业务价值

现在团队手动查价、截图、填表。问题不是效率低这么简单。

更大的问题是销售现场缺少一个可信、可复核、可留痕的展示闭环。

第一版要解决三件事：

1. 让销售能在客户面前展示同一航班三平台对比。
2. 让客户会后拿到 Excel 可以复核。
3. 让内部保留截图证据，避免价格争议时说不清。

### 2.2 用户核心路径

第一版使用者只有两类：

- 用户本人：使用管理采集端，负责手动触发更新、登录平台、生成当前批次数据、导出 Excel 或离线演示包。
- 销售：使用离线销售演示包，在客户面前点击演示页，展示随机同一航班在 3 个不同平台的价格，完成销售任务。

用户本人只负责三件事：

1. 登录青猫差旅、携程商旅、阿里商旅三个平台。
2. 点击“开始采集”。
3. 采集完成后点击“导出”。

其余工作由系统完成。

管理采集端显示最近一次采集状态：

- 已有可用演示数据
- 暂无可用演示数据
- 最近一次采集时间
- 成功数量
- 失败数量

如果需要更新，用户本人在管理采集端点击“开始采集”。

系统按固定航线逐条处理：

1. 选择未来 3 天后的日期。
2. 打开青猫差旅搜索结果。
3. 在青猫候选池中随机抽取一个经济舱航班。
4. 国内只抽直飞。
5. 国际优先直飞，无直飞时允许中转。
6. 用同一个航班号去携程商旅、阿里商旅查询。
7. 记录价格、退改、行李、是否可订。
8. 保存三平台截图证据。
9. 生成对比结论。
10. 生成 Excel。
11. 生成离线销售演示包。

完成后：

- 系统生成当前批次离线销售演示包数据。
- Excel 可导出。
- 离线销售演示包可导出。
- 后台或 Excel 留痕页保存完整采集字段。

销售外出演示时，只打开离线销售演示包里的演示页。这个页面只展示已发布的当前批次数据，不能触发采集，也不能依赖用户本人电脑在线。

### 2.3 客户看到什么

客户现场只看核心信息：

- 出发地-目的地
- 出行日期
- 航班号
- 航司
- 直飞/中转
- 青猫差旅价格
- 携程商旅价格
- 阿里商旅价格
- 青猫优势金额，或贵了多少
- 简短结论
- 查看证据

不要把采集批次、采集人、URL、后台字段堆给客户。

也不要在客户现场展示“开始采集”动作。

### 2.4 销售话术边界

前台展示强调“系统随机抽取航班”。

不强调候选池来自青猫差旅。

候选池来源作为后台采集口径和审计记录保留。

原因很现实：客户关心的是三平台同屏对比，不需要听内部抽样机制。但系统要能解释随机规则，避免客户质疑销售手动挑样本。

### 2.5 Edge Cases

- 青猫候选池没有航班：该航线记录为本次无法采样。
- 携程商旅或阿里商旅没有同一航班：记录“未展示/不可订/无同航班”，不能换另一个航班。
- 国际航线无直飞：允许中转，但必须记录中转城市和总时长。
- 页面登录过期：提示用户重新登录，不自动重试账号密码。
- 平台页面结构变化：采集失败时保留错误截图和失败原因。
- 三平台价格单位或税费口径不同：第一版先按页面展示价格记录，同时截图留证。
- 平台阻止 headless 后台采集：切换到 headed 浏览器非置顶或可最小化运行，仍然不使用 OCR 作为主路径。

## 4. Eng Review

### 4.1 建议模块

第一版建议拆成 7 个模块。

#### 模块 1：项目基础与数据模型

职责：

- 建立航线清单配置。
- 建立采集批次模型。
- 建立航班样本模型。
- 建立平台价格记录模型。
- 建立截图证据模型。

核心对象：

- Route：固定航线
- CollectionBatch：手动触发的采集批次
- FlightSample：随机抽中的航班
- PlatformQuote：某平台对该航班的报价
- Evidence：页面截图和来源信息

#### 模块 2：人工登录后的浏览器采集

职责：

- 使用已登录浏览器状态。
- 打开青猫差旅、携程商旅、阿里商旅。
- 读取页面信息。
- 保存截图证据。
- 优先后台静默采集，不让浏览器窗口长期置顶影响用户其他工作。
- 主路径不使用 OCR；优先读取网页 DOM、页面元素、接口返回数据。
- 截图只作为证据，不作为主要采集数据来源。

第一版不做账号密码登录。

#### 模块 3：随机抽样逻辑

职责：

- 对固定航线查询未来 3 天后的结果。
- 从青猫候选池中随机抽航班。
- 国内过滤直飞。
- 国际优先直飞，无直飞时允许中转。
- 生成可追溯的随机结果。

随机结果需要记录，但不在客户页强调候选池来源。

#### 模块 4：比价与结论生成

职责：

- 对三平台价格做结构化比较。
- 找出最低价平台。
- 计算青猫与最低价差额。
- 生成短结论。

结论口径：

- 青猫价格最低：青猫价格有优势，较其他平台低 X-Y 元。
- 青猫价格持平：三平台价格基本持平。
- 青猫非最低：青猫价格暂不占优，较最低价高 X 元。
- 平台无同航班：该平台未展示同一航班，不参与最低价比较，但保留状态。

#### 模块 5：管理采集端

职责：

- 展示最近一次采集状态。
- 提供“开始采集”按钮。
- 展示采集进度、成功数量、失败数量。
- 提供 Excel 导出入口。
- 生成当前批次离线销售演示包。

设计原则：

- 面向内部人员。
- 允许展示采集状态和失败原因。
- 不给客户现场使用。

#### 模块 6：离线销售演示包

职责：

- 展示当前批次已发布数据状态。
- 展示国内/国际结果。
- 三平台价格同屏展示。
- 提供“查看证据”入口。
- 不提供“开始采集”按钮。
- 不触发任何平台采集动作。
- 可复制到销售自己的电脑上离线打开。
- 包含演示页、结构化数据、证据截图索引。

设计原则：

- 少字段。
- 强对比。
- 客户能一眼看懂青猫是否有优势。
- 不展示内部采集噪音。
- 必须美观，适合客户现场观看。
- 视觉重点放在三平台价格对比、青猫优势金额、证据入口。

#### 模块 7：Excel 导出

职责：

- 导出 Excel 汇总页。
- 导出或引用证据索引。
- 导出内部留痕页。

Excel 至少包含：

- 汇总页
- 证据索引页
- 留痕页

Excel 美观要求：

- 版式清晰。
- 标题明确。
- 表头固定。
- 列宽合理。
- 重点价格和结论突出。
- 适合会后发送给客户。

## 6. 实施顺序

### 阶段 1：可运行骨架

目标：

- 建立本地项目结构。
- 建立管理采集端框架。
- 建立离线销售演示包框架。
- 使用假数据跑通展示。
- 跑通 Excel 导出。

验收：

- 页面能显示 20 条样例。
- Excel 能导出。
- 不接真实平台。

### 阶段 2：浏览器采集验证

目标：

- 启动 GStack Browser 或等价可控浏览器。
- 用户人工登录三平台。
- 对 1 条国内航线做真实采集。
- 保存截图证据。
- 验证能否后台静默运行。
- 验证是否可以不用 OCR 读取价格、退改、行李字段。

验收：

- 能读取青猫候选航班。
- 能随机抽 1 个航班。
- 能在携程商旅和阿里商旅查同一航班。
- 能保存三平台截图。
- 优先方案为后台静默采集。
- 如果后台静默失败，必须明确失败原因和兜底方式。

### 阶段 3：扩展到 20 条航线

目标：

- 国内 10 条、国际 10 条全部跑通。
- 失败状态可记录。
- 采集完成后生成当前批次结果。

验收：

- 每条航线有成功结果或明确失败原因。
- 失败不阻断整个批次。
- 截图证据完整。

### 阶段 4：演示页与 Excel 打磨

目标：

- 演示页适合客户现场看。
- Excel 适合会后发送。
- 留痕页适合内部复盘。

验收：

- 客户页不显示内部字段。
- Excel 有汇总页、证据索引页、留痕页。
- 管理端最近一次采集状态清晰。
- 离线销售演示包视觉上适合客户现场展示。
- Excel 视觉上适合会后发送给客户。

## 7. 技术风险

### 7.1 最大风险：平台页面和登录状态

三家商旅平台可能有：

- 验证码
- 企业 SSO
- 登录过期
- 页面结构变化
- 反自动化限制

第一版用人工登录规避账号密码自动登录风险。

### 7.4 后台静默采集风险

部分平台可能检测 headless 浏览器，或要求页面处于可见浏览器环境。

处理方式：

- 优先验证 headless 后台采集。
- 如果失败，验证 headed 浏览器非置顶或可最小化采集。
- 不以 OCR 作为主路径。
- 截图用于证据，不用于主要字段识别。

### 7.2 第二风险：同一航班不一定三平台都有

这不是系统错误。

必须记录为业务事实：

- 未展示
- 不可订
- 无同航班

不能替换成另一个航班。

### 7.3 第三风险：价格口径不一致

平台可能展示不同价格口径：

- 票面价
- 含税价
- 企业协议价
- 服务费后价格

第一版按页面展示价格记录，并保存截图。

## 8. 质量门禁

进入开发后，每个模块完成都要检查：

- 没有静默失败。
- 失败有用户可见提示。
- 每个 loading 状态都有成功或失败状态。
- 空结果有空状态。
- 每个按钮点击有反馈。
- 不保留 console.log。
- 不在代码、配置、文档中保存账号密码。
- 不直接修改原始 Excel 模板。

## 9. 第一版通过标准

第一版完成时，必须满足：

1. 用户可以看到最近一次采集状态。
2. 用户本人可以在管理采集端手动点击开始采集。
3. 系统能按 20 条固定航线生成结果。
4. 每条航线随机抽取 1 个未来 3 天后的经济舱航班。
5. 三个平台对比同一航班。
6. 国内只采直飞。
7. 国际优先直飞，无直飞时允许中转。
8. 三平台截图证据被保存。
9. 离线销售演示包可给客户现场看。
10. 离线销售演示包不显示“开始采集”按钮。
11. 销售可在自己的电脑上离线打开演示包查看当前批次演示信息。
12. 销售演示页视觉上适合客户现场展示。
13. Excel 可导出，包含汇总页、证据索引页、留痕页。
14. Excel 视觉上适合会后发送给客户。
15. 采集优先后台静默运行，不长期占用前台窗口。
16. 主路径不使用 OCR 识别屏幕文字。

## 10. 下一步决策

以上产品与技术方案是否通过？

确认通过后，再进入开发。开发时先做阶段 1：可运行骨架，不直接碰真实平台采集。

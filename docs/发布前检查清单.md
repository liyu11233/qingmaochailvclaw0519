# 发布前检查清单

适用于把当前项目提交到 GitHub 前的最后检查。

## 1. 范围确认

- [ ] 本次改动只包含预期文件。
- [ ] 没有改采集核心逻辑，除非本次任务明确要求。
- [ ] 没有删除现有代码和本地输出文件。
- [ ] 没有把真实账号、登录态、Cookie、截图、导出 Excel、销售长截图写进提交。

## 2. 本地运行确认

- [ ] 可以启动管理端。
- [ ] 管理端地址可以打开。
- [ ] `打开登录浏览器` 可打开三平台入口。
- [ ] 人工登录后，`连接当前登录窗口` 能识别三平台。
- [ ] 完整真实采集完成后，可以导出 Excel。
- [ ] 完整真实采集完成后，可以导出销售长截图。

## 3. 自动化检查

```bash
npm test
npm run typecheck
npm run build
```

三条命令都必须通过。

## 4. Git 忽略检查

```bash
git check-ignore -v outputs .runtime .env node_modules
```

这些路径必须被忽略。

再检查暂存内容：

```bash
git status --short
git diff --stat
git diff --cached --stat
```

不要提交以下内容：

- `outputs/`
- `.runtime/`
- `.env`
- `node_modules/`
- `dist/`
- `screenshots/`
- `playwright/.auth/`
- 真实采集截图
- 导出 Excel
- 销售长截图 PNG

## 5. Git LFS 检查

根目录两个原始 Excel 模板应由 Git LFS 管理：

```bash
git lfs ls-files
```

预期能看到：

- `更新版-TMC价格对比.xlsx`
- `更新版-青猫商旅与OTA对比.xlsx`

如果 `.xlsx` 被当成普通大文件提交，先停止，不要 push。

## 6. 提交前最后确认

- [ ] README 能让未来的自己从零启动。
- [ ] `.env.example` 只包含示例配置，不含真实账号信息。
- [ ] `.command` 文件可双击启动。
- [ ] 本次提交没有包含不相关代码改动。
- [ ] 用户确认后再推送 GitHub。
